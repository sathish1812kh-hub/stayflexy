# V5.2 Unified Autonomous Software Intelligence Orchestrator

## Purpose

This document defines the mandatory workflow for all development activities.

The objective is to:

- Prevent duplicate feature development
- Prevent duplicate package installation
- Preserve project memory across sessions
- Maintain complete traceability
- Detect impact before implementation
- Predict consequences before implementation
- Maintain Neo4j, Graphiti, and GraphQL synchronization
- Prevent architecture drift

---

# Core Rules

## Rule 1

Chat history is NOT the source of truth.

## Rule 2

Codebase reality overrides assumptions.

## Rule 3

Runtime reality overrides static assumptions.

## Rule 4

No implementation without impact analysis.

## Rule 5

No deployment without rollback validation.

## Rule 6

No task completion without synchronization.

- Build Passed

- Deployment Validated

- Graphiti Updated
- GraphQL Updated

# Startup Workflow

Load:

1. current-state.md (Verify Git status/hash mismatch)
2. project-context.md
3. active-tasks.md
4. decision-log.md
5. Neo4j Snapshot (Probed JIT during query phase)
6. Graphiti Memory

Then:

Build Session Context (Verify against dirty git status tree)

Then:

Resume Development

---

# Non-Negotiable Rule

The next task must never start from stale knowledge.

Every completed task must leave:

- Neo4j Current
- Graphiti Current
- GraphQL Current
- Project Memory Updated

Only then may the next task begin.
